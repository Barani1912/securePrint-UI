# 🛡️ SecurePrint UI

SecurePrint is a privacy-first, WebRTC-powered Peer-to-Peer (P2P) file transfer application designed specifically for secure document printing. It eliminates the need for intermediate servers, ensuring your sensitive documents never leave your browser's memory until they reach the recipient.

---

## 🚀 Key Features

- **P2P Transfer:** Direct browser-to-browser connection using WebRTC (PeerJS).
- **Zero Data Retention:** Files are processed entirely in RAM. No database, no cloud storage, no logs.
- **PIN-Protected Access:** The recipient must enter a 4-digit PIN generated by the sender to "unlock" the document.
- **Chunked Data Streaming:** Large documents (up to 10MB) are split into 16KB chunks to ensure stability over unreliable connections.
- **Auto-Wipe:** Instant memory cleanup and Blob revocation once the session ends or PIN attempts fail.
- **Multilingual Support:** Built with `i18next` for global accessibility.

## 🚦 Getting Started

### Prerequisites
- Node.js (v18 or higher)
- npm or yarn

### Installation
1. Navigate to the UI directory:
   ```bash
   cd securePrint-UI
   ```
2. Install dependencies:
   ```bash
   npm install
   ```

### Running Locally
To start the development server:
```bash
npm run dev
```

The application will be available at `http://localhost:5173`. Make sure the **SecurePrint Backend** is also running for peer signaling.

---

## 🔒 Security Lifecycle
1. **Handshake:** Establish a direct tunnel via the signaling server.
2. **Transfer:** Stream encrypted chunks directly to the recipient's RAM.
3. **Lock:** The file is received but stays hidden behind a PIN challenge.
4. **Wipe:** Successful print or failed PIN attempt triggers a mandatory memory purge.
